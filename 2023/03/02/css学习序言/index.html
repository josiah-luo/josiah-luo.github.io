2. css选择器

作用：选择页面上的某一个或者某一类元素

2.1基本选择器

  1. 标签选择器：选择一类标签 标签{}

  2. 类选择器 class：选择所有class属性一致的元素，跨标签 .类名{}

  1. id选择器（id不能够以数字开头！）： #id名{}

优先级：id>class>标签

2.2层次选择器

  1. 后代选择器：在某个元素的后面 祖爷爷 爷爷 爸爸 你 (后面的元素都会变)

    /在body加空格加需要更改的元素 body p/

  2. 子选择器，一代，儿子，只有一代

  3. 相邻（兄弟）选择器 同辈 一定要相邻 只有一个下面的才改变，也就是向下选择器

  4. 通用选择器: 当前选中元素向下的所有兄弟元素

2.3 结构 伪类选择器

选择太复杂，不深入学习，平时使用id和class即可

2.4 属性选择器（常用）

本质就是首先选中某个标签，然后中括号标签里面的含有的某个属性（id，class……）

本质：将id和class互相结合

重点：掌握标签[属性]{

} eg：a[id*=111]{ background-color:red; }

3.4 超链接伪类

CSS为一些特殊效果准备了特定的工具，我们称之为“伪类”。其中有几项是我们经常用到的，下面我们就详细介绍一下经常用于定义链接样式的四个伪类，它们分别是：

  • :link

  • :visited

  • :hover

  • :active

    因为我们要定义链接样式，所以其中必不可少的就是超级链接中的锚标签–a，锚标签和伪类链接起来书写的方法就是定义链接样式的基础方法，它们的写法如下：

  • a:link，定义正常链接的样式

  • a:visited，定义已访问过链接的样式

  • a:hover，定义鼠标悬浮在链接上时的样式

  • a:active，定义鼠标点击链接时的样式

    定义A链接样式的正确的顺序：a:link、a:visited、a:hover、a:active

5.浮动

5.1 标准文档流

块级元素 ：独占一行

h1~h6 p div 列表……

行内元素：不独占一行

span a img strong……

行内元素可以被包含在块级元素中，反之，则不可以

5.2 display

block 块元素
inline 行内元素
inline-block 是块元素的同时保留了行内元素的能够并列的特性，能够内联，在一行
none 让元素消失

重点使用：block和inline-block

其中inline-block可以用作导航页的横板排列

这也是一种实现行内元素排列的方式，但是我们很多情况都是用float

5.3 浮动

  1. 左右浮动 float

5.4 父级边框塌陷问题

clear

/*

clear:right 右侧不允许有浮动

clear:left 左侧不允许有浮动

clear:both 两侧不允许有浮动

clear:none

*/

解决方案

  1. 增加父级元素的高度
  1. 增加一个空的div标签，清除浮动
  1. overflow

在父级元素中增加一个 overfloat：hidden；

  1. 父类添加伪类：after

#father:after{

content:’’;

display: block;

clear:both;

}

小结：

  1. 浮动元素后面增加空div 简单，代码中尽量避免空div
  2. 设置父级元素的高度 简单 元素如果有固定高度，就会被限制
  3. overflow 简单，下拉的一些场景，不能使用，尽量避免使用
  4. 父类添加伪类 ：after （推荐） 写法稍微复杂一点，但是没有任何副作用，推荐使用

5.5 对比

    方向不可以控制

  • float

    浮动起来的话会脱离标准文档流，所以要解决父级边框塌陷问题
